Ibland får vi följande fråga: varför ska vi använda Aspose-produkter istället för xlsx4j? Denna fråga är lätt att besvara: funktioner och funktionalitet.
{{% /alert %}}


## **xlsx4j**
xlsx4j är ett öppen källkod (ASLv2) Java-bibliotek för att skapa och manipulera Microsoft Open XML Excel xlsx-filer liknande docx4j för att manipulera Microsoft Open XML Word docx-filer.
Det är liknande Microsofts OpenXML SDK, men för Java. docx4j använder JAXB för att skapa in-memory objektrepresentation.
Du måste ta dig tid att förstå JAXB och filstrukturen för Open XML.

Läs mer på [docx4java.org](https://www.docx4java.org/trac/docx4j)
## **Aspose.Cells for Java**
Aspose.Cells for Java är en prisbelönt Excel-kalkylbladskomponent som tillåter Java-utvecklare att bädda in förmågan att läsa, skriva och manipulera Excel®-kalkylblad (XLS, XLSX, XLSM, XLSB, XLTX, SpreadsheetML, CSV, ODS), HTML, MHTML, PDF och bildfilformat i sina egna Java-applikationer utan att behöva förlita sig på Microsoft Excel®.

Aspose.Cells for Java är en mogen, skalbar och funktionsspäckad komponent som erbjuder många funktioner som är långt bortom de enkla dataexporteringsförmågorna hos andra leverantörer. Med Aspose.Cells for Java kan utvecklare exportera data, formatera kalkylblad på mest granulära nivå, importera bilder, skapa diagram, tillämpa och beräkna komplexa formler, strömma Excel®-data, spara i olika format och mycket mer - allt detta utan behov av Microsoft Excel® eller Microsoft Office Automation.

Läs mer på [aspose.com](https://products.aspose.com/cells/java/)

## **Varför inte xlsx4j**
xlsx4j kan endast parsa och spara Microsoft Open XML Excel xlsx-filer,Aspose.Cells for Java kan behandla alla Excel-filformat (XLS, XLSX, XLSM, XLSB, XLTX, SpreadsheetML, CSV, ODS) och stöder olika filformatkonverteringar.

## **Aspose.Cells Java for xlsx4j**
Projekt Aspose.Cells för xlsx4j visar hur olika uppgifter kan utföras med hjälp av Aspose.Cells Java APIs jämfört med xlsx4j. Projektet täcker även funktioner som endast är tillgängliga i Aspose.Cells APIs men inte i xlsx4j för att hantera kalkylblad.

Detta projekt är till hjälp för utvecklare som vill jämföra xlsx4j med Aspose.Cells ELLER migrera från xlsx4j till Aspose.Cells.
## **Systemkrav och stödda plattformar**
### **Systemkrav**
Följande är systemkraven för att köra Aspose.Cells Java för xlsx4j:

- Java 1.4 eller högre installerat.
- Nerladdad Aspose.Cells-komponent.
- Nerladdade docx4j-bibliotek.
### **Stödda plattformar**
Följande är de stödda plattformarna:

- Aspose.Cells 8.3.0 eller högre.
- docx4j 3.1.0 eller högre bibliotek.
- Java IDE (Eclipse, NetBeans, IntelliJ, ...).

Följande versioner av körnings exempel är tillgängliga för nedladdning på alla nedan angivna sociala kodningssajter:

### **GitHub**
- [Aspose.Cells Java for Xlsx4j - v 1.0](https://github.com/aspose-cells/Aspose.Cells-for-Java/releases/tag/Aspose.Cells_Java_for_Xlsx4j-v1.0.0)

För att konfigurera **Aspose.Cells for Java API** inkludera Aspose-repository URL:en och Aspose.Cells for Java API maven-biblioteket i din **pom.xml** som följande:
### **Aspose Maven Repository**
{{< highlight java >}}

 <repositories>

    <repository>

        <id>aspose-maven-repository</id>

        <name>Aspose Maven Repository</name>

        <url>http://repository.aspose.com/repo/</url>

    </repository>

</repositories>

{{< /highlight >}}
### **Aspose.Cells for Java Maven Dependency**
{{< highlight java >}}

 <dependencies>

    <dependency>

        <groupId>com.aspose</groupId>

        <artifactId>aspose-cells</artifactId>

        <version>8.6.0</version>

    </dependency>

</dependencies>

{{< /highlight >}}

**Obs:** Använd den tillgängliga senaste versionen av Aspose API.
