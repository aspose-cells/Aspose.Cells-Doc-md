## **Vill du påskynda utvecklingen?**

{{% alert color="primary" %}}

Du kan använda Aspose.Cells för att skapa Excel-filer från grunden med hjälp av dess flexibla API. En av de vanligaste användningsområdena för Aspose.Cells är att skapa och fylla i Excel-filer med data, som kan komma från en befintlig databas, en användare eller andra källor, dynamiskt. Förutom att lägga till data i Excel-filer måste du också skriva för att tillämpa formatering (utseende och känsla).

Formatering tar tid. Vad skulle kunna påskynda utvecklingen?

{{% /alert %}}

## **Lösningen - Designer-kalkylblad**

För att påskynda din utveckling erbjuder Aspose.Cells det spännande kalkylbladet för funktionsdesigner.

### **Vad är ett designerkalkylblad?**

Ett designerkalkylblad är en Excel-fil skapad med Microsoft Excel där all formatering redan är tillämpad och innehållet kan bestämmas vid designtillfället. Det är mer som en mallfil som inte innehåller några data utan bara formateringen, som visas nedan:

**Designer kalkylblad**

![todo:image_alt_text](what-is-a-designer-spreadsheet_1.png)

### **Hur man använder det?**

 När du har skapat ett designerkalkylblad, använd Aspose.Cells för att läsa designerkalkylarket och lägga till data till dess angivna positioner (enligt de fördefinierade formatinställningarna) vid körning. Till exempel, i skärmdumpen ovan, kan du se att vi redan vid designtillfället bestämde oss för att lägga till**produktnamn**, **kategori namn**, **Kvantitet per enhet** och**Enheter i lager**kolumner. De har inga data ännu: detta läggs till vid körning. Ovanstående figur visar ett mycket enkelt kalkylblad men designerkalkylblad sparar tid även på komplexa kalkylblad.

### **Smarta markörer**

 Dessutom låter Aspose.Cells dig lägga till[smarta markörer](/cells/sv/python-net/smart-markers/) till designerkalkylarket. Smarta markörer bearbetas under körning med WorkbookDesigner-klassens processmetod. Detta kan hjälpa dig att skriva mindre kod och ändå få bra resultat.

[Ta reda på mer om smarta markörer](/cells/sv/python-net/smart-markers/).
