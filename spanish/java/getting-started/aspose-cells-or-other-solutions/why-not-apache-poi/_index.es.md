## **Apache POI SS - HSSF y XSSF**
**HSSF** es la implementación pura en Java del proyecto POI del formato de archivo de Excel '97(-2007).
**XSSF** es la implementación pura en Java del proyecto POI del formato de archivo Excel 2007 OOXML (.xlsx).

HSSF y XSSF proporcionan formas de leer hojas de cálculo, crear, modificar, leer y escribir hojas de cálculo XLS. Ofrecen:

- estructuras de bajo nivel para aquellos con necesidades especiales
- una API de modelo de eventos para un acceso eficiente de solo lectura
- una API de modelo de usuario completo para crear, leer y modificar archivos XLS

Leer más en [poi.apache.org](http://poi.apache.org/components/spreadsheet/index.html)
## **Aspose.Cells for Java**
Aspose.Cells for Java es un componente galardonado de hoja de cálculo de Excel que permite a los desarrolladores de Java incrustar la capacidad de leer, escribir y manipular hojas de cálculo de Excel® (XLS, XLSX, XLSM, XLSB, XLTX, SpreadsheetML, CSV, ODS), HTML, MHTML, PDF y formatos de archivos de imagen en sus propias aplicaciones Java sin depender de Microsoft Excel®.

Aspose.Cells for Java es un componente maduro, escalable y rico en funciones que ofrece muchas funciones que van más allá de las simples capacidades de exportación de datos de otros proveedores. Con Aspose.Cells for Java, los desarrolladores pueden exportar datos, formatear hojas de cálculo hasta el nivel más granular, importar imágenes, crear gráficos, aplicar y calcular fórmulas complejas, transmitir datos de Excel®, guardar en varios formatos y mucho más, todo esto sin necesidad de Microsoft Excel® ni de Automatización de Office de Microsoft.

Lee más en [aspose.com](https://products.aspose.com/cells/java/)

## **¿Por qué no Apache POI SS - HSSF y XSSF**

Vale la pena señalar que algunas tareas se pueden lograr con Aspose.Cells, pero no se pueden lograr con Apache POI. Por ejemplo, si necesita convertir archivos de Excel a Pdf, JSON e imágenes, entonces no puede usar solo Apache POI, también necesita Microsoft Excel 365 u otras herramientas.

Puede comparar POI con Aspose.Cells Para hacer esto, le sugerimos que se familiarice con el proyecto Aspose.Cells for POI (HSSF y XSSF) - muestra cómo se pueden realizar diferentes tareas utilizando la API Aspose.Cells para .Java versus POI. El proyecto también cubre características para trabajar con documentos de texto que solo están disponibles en Aspose.Cells, pero no en Apache POI.

Este proyecto también es útil para desarrolladores que buscan migrar de POI a Aspose.Cells.

### **Cómo configurar el código fuente**
Por favor, siga estos sencillos pasos para abrir y extender el código fuente al usar:
#### **Eclipse IDE**
1. Descargar/Clonar el código fuente.
1. Abra Eclipse y elija **Archivo > **Importar...**
1. Seleccione **General** > **Proyectos Existentes en Espacio de Trabajo**.
1. Vaya a **seleccionar el directorio raíz** del **último código fuente** que haya descargado.
1. Haga clic en **Finalizar** para importar con éxito el proyecto descargado.
#### **NetBeans IDE**
1. Descargar/Clonar el código fuente.
1. Abra NetBeans y elija **Archivo > **Nuevo Proyecto...**
1. En la pestaña **Elegir Proyecto**: Seleccione **Java** > **Proyecto Java con Fuentes Existentes**.
1. En la pestaña **Nombre y Ubicación**: Proporcione su **Nombre de Proyecto** deseado y **Carpeta de Proyecto**.
1. En la pestaña **Fuentes Existentes**: Haga clic en **Agregar Carpeta...** y vaya a seleccionar el directorio raíz del **último código fuente** que haya descargado.
1. Haga clic en **Finalizar** para crear con éxito el proyecto descargado.
