{{% alert color="primary" %}}

Наиболее вероятный сценарий заключается в том, что вы используете Aspose.Cells для преобразования электронных таблиц в PDF. Если вы делаете это в операционной системе, отличной от Windows, такой как Linux, то в этом разделе объясняется, как убедиться, что Aspose.Cells отображает ваши электронные таблицы с максимальной точностью.

Чтобы убедиться, что электронные таблицы, преобразованные с помощью Aspose.Cells, выглядят как можно ближе к оригиналу, вам может потребоваться установить «Windows шрифты» или «шрифты TrueType» в вашей системе Linux, поскольку наиболее часто используемые шрифты TrueType не поставляются предварительно установленными. Дистрибутивы Linux по умолчанию.

Есть два основных способа получить шрифты TrueType в системе Linux:

1. Скопируйте файлы шрифтов (.TTF и .TTC) с компьютера Windows на свой компьютер с Linux.
1. Установите пакет шрифтов TrueType, например msttcorefonts.

{{% /alert %}}

## **Скопируйте шрифты с машины Windows**

Простой и быстрый способ — скопировать файлы .TTF и .TTC из каталога C:\Windows\Fonts на компьютере Windows в какой-либо каталог на вашем компьютере с Linux. Вам не нужно каким-либо образом устанавливать или регистрировать эти шрифты в Linux, вам просто нужно указать расположение шрифтов с помощью метода FontConfigs.setFontFolder в вашем приложении.

{{% alert color="primary" %}}

Насколько мы можем судить, Microsoft лицензирует шрифты для свободного использования всеми, но, пожалуйста, проверьте лицензирование шрифтов для себя.

{{% /alert %}}

## **Установите пакет шрифтов TrueType**

Приведенная ниже информация поможет вам шаг за шагом установить самые известные шрифты TrueType Microsoft в дистрибутивах Linux, таких как Fedora и Red Hat Enterprise Linux (RHEL).

{{% alert color="primary" %}}

Вам могут потребоваться привилегии уровня «root», поэтому войдите в систему как «root» или настройте sudo.

{{% /alert %}}

Вот как это сделать с помощью терминала.

1. Убедитесь, что у вас установлены следующие пакеты RPM.
   1. **rpm-сборка**: Если он не установлен, используйте следующую команду для установки пакета rpm-build.

{{< highlight "java" >}}

yum install rpm-build cabextract ttmkfdir

{{< /highlight >}}

1. **wget**: Если не установлен, используйте следующую команду

{{< highlight "java" >}}

yum \-y install wget

{{< /highlight >}}

1. Загрузите последний файл спецификации msttcorefonts из SourceForge, используя следующую команду:

{{< highlight "java" >}}

wget http://corefonts.sourceforge.net/msttcorefonts-2.5-1.spec

{{< /highlight >}}

1. Создайте файл RPM, используя ранее загруженный файл спецификаций и следующую команду:

{{< highlight "java" >}}

rpmbuild \-ba msttcorefonts-2.5-1.spec

{{< /highlight >}}

1. Файл RPM будет храниться в: /root/rpmbuild/RPMS/noarch/, установите его следующим образом:

{{< highlight "java" >}}

rpm \-ivh /root/rpmbuild/RPMS/noarch/msttcorefonts-2.5-1.noarch.rpm

{{< /highlight >}}

1. Перезагрузите машину, чтобы изменения вступили в силу.

Приведенные выше инструкции позволят установить пакет TTF Microsoft, включающий следующие семейства шрифтов:

1. Андейл Mono
1. Arial Black/Arial (полужирный, курсив, полужирный курсив)
1. Comic Sans MS (жирный шрифт)
1. Courier New (жирный, курсив, полужирный курсив)
1. Джорджия (жирный, курсив, полужирный курсив)
1. Влияние
1. Тахома
1. Times New Roman (жирный, курсив, полужирный курсив)
1. Требушет (полужирный, курсив, полужирный курсив)
1. Verdana (жирный, курсив, полужирный курсив)
1. Вебдинги

 В Ubuntu перейдите в диспетчер пакетов Synaptic. Найдите и установите**ttf-mscorefonts-установщик** упаковка.
