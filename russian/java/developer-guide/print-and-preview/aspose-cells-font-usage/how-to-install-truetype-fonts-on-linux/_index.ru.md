{{% alert color="primary" %}}

Наиболее вероятный сценарий - вы используете Aspose.Cells для преобразования электронных таблиц в PDF. Если вы делаете это в операционной системе, не основанной на Windows, такой как Linux, то в этой теме объясняется, как обеспечить, чтобы Aspose.Cells отображал ваши электронные таблицы с наилучшей точностью.

Чтобы убедиться, что электронные таблицы, преобразованные Aspose.Cells, по возможности максимально близки к оригиналу, вам может потребоваться установить "шрифты Windows" или "шрифты TrueType" на вашу систему Linux, потому что наиболее часто используемые шрифты TrueType не предустановлены в дистрибутивах Linux по умолчанию.

Существуют два основных способа получения шрифтов TrueType на системе Linux:

1. Скопируйте файлы шрифтов (.TTF и .TTC) с Windows-машины на вашу машину с Linux.
1. Установите пакет шрифтов TrueType, такие как msttcorefonts.

{{% /alert %}}

## **Копирование шрифтов с Windows-машины**

Простой и быстрый способ - скопировать файлы .TTF и .TTC из каталога C:\\Windows\\Fonts на Windows-машине в какой-либо каталог на вашей машине с Linux. Вам не нужно устанавливать или регистрировать эти шрифты на Linux никаким образом, вам просто нужно указать расположение шрифтов с использованием метода FontConfigs.setFontFolder в вашем приложении.

{{% alert color="primary" %}}

Насколько мы можем судить, Майкрософт лицензирует шрифты для свободного использования, но, пожалуйста, проверьте лицензирование шрифтов самостоятельно.

{{% /alert %}}

## **Установите пакет шрифтов TrueType**

Нижеприведенная информация будет пошагово руководить вас по установке самых известных в мире TrueType шрифтов Microsoft на дистрибутивах Linux, таких как Fedora и Red Hat Enterprise Linux (RHEL).

{{% alert color="primary" %}}

Вам может потребоваться права 'root', поэтому войдите как 'root' или настройте sudo.

{{% /alert %}}

Вот как это сделать с помощью терминала.

1. Убедитесь, что у вас установлены следующие пакеты RPM.
   1. **rpm-build**: Если не установлен, воспользуйтесь следующей командой для установки пакета rpm-build.

{{< highlight java >}}

yum install rpm-build cabextract ttmkfdir

{{< /highlight >}}

1. **wget**: Если не установлен, воспользуйтесь следующей командой.

{{< highlight java >}}

yum \-y install wget

{{< /highlight >}}

1. Загрузите последний файл спецификации msttcorefonts с сайта SourceForge с помощью следующей команды.

{{< highlight java >}}

wget http://corefonts.sourceforge.net/msttcorefonts-2.5-1.spec

{{< /highlight >}}

1. Создайте файл RPM с помощью ранее загруженного файла спецификации и следующей команды.

{{< highlight java >}}

rpmbuild \-ba msttcorefonts-2.5-1.spec

{{< /highlight >}}

1. Файл RPM будет сохранен в: /root/rpmbuild/RPMS/noarch/, установите его следующим образом.

{{< highlight java >}}

rpm \-ivh /root/rpmbuild/RPMS/noarch/msttcorefonts-2.5-1.noarch.rpm

{{< /highlight >}}

1. Перезапустите машину, чтобы внесенные изменения вступили в силу.

Предоставленные выше инструкции установят пакет Microsoft TTFs, включая следующие семейства шрифтов:

1. Andale Mono
1. Arial Black/Arial (Жирный, Курсив, Жирный курсив)
1. Comic Sans MS (Жирный)
1. Courier New (Жирный, Курсив, Жирный курсив)
1. Georgia (Жирный, Курсив, Жирный курсив)
1. Impact
1. Tahoma
1. Times New Roman (Жирный, Курсив, Жирный курсив)
1. Требушет (Жирный, Курсив, Жирный курсив)
1. Verdana (Полужирный, Курсив, Полужирный курсив)
1. Webdings

{{% alert color="primary" %}}

На Ubuntu перейдите в Менеджер пакетов Synaptic. Найдите и установите пакет **ttf-mscorefonts-installer**.

{{% /alert %}}
