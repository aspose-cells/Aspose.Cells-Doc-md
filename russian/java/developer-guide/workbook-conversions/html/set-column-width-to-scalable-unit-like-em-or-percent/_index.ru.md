Создание файла HTML из электронной таблицы очень распространено. Размер столбцов определяется в «pt», что работает во многих случаях. Однако может быть случай, когда этот фиксированный размер может не потребоваться. Например, если ширина панели контейнера составляет 600 пикселей, где отображается эта страница HTML. В этом случае вы можете получить горизонтальную полосу прокрутки, если ширина сгенерированной таблицы больше. Требовалось, чтобы этот фиксированный размер был изменен на масштабируемую единицу, такую как em или процент, чтобы получить лучшее представление. Следующий пример кода можно использовать там, где[**HtmlSaveOptions.WidthScalable**](https://reference.aspose.com/cells/java/com.aspose.cells/htmlsaveoptions#WidthScalable) установлен на**истинный** для создания масштабируемой ширины.
