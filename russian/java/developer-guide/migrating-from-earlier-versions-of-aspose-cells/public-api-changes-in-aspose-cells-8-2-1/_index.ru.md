Этот документ описывает изменения в общедоступном API Aspose.Cells от версии 8.2.0 до 8.2.1, которые могут быть интересны разработчикам модулей/приложений. В нем содержится информация не только о новых и обновленных общедоступных методах, но и описания любых изменений в поведении за кулисами в Aspose.Cells.

{{% /alert %}} 
## **Добавлен метод getValidation() для класса Cell**
Валидация данных - одна из функций, которые используют разработчики таблиц для предотвращения вставки недопустимых значений в определенную ячейку. В версии Aspose.Cells for Java 8.2.1 API предоставил простой механизм, который определяет, была ли применена валидация данных к ячейке. Используйте метод getValidation класса Cell для получения любой примененной валидации. Если нет валидации, метод возвращает null. Аналогично, вы можете использовать метод getValidationInCell класса ValidationCollection для получения примененной валидации к любой ячейке, указав индексы ее строки и столбца.

{{% alert color="primary" %}} 

Пожалуйста, проверьте подробную статью о [Получении валидации, примененной к ячейке](https://docs.aspose.com/cells/java/get-validation-applied-on-a-cell/) для получения дополнительной информации.

## **Добавлен метод getValidationValue() для класса Cell**
Кроме определения того, была ли применена валидация, можно также проверить, удовлетворяет ли заданное значение правилам валидации данных для определенной ячейки. Эта функция полезна в ситуациях, когда требуется проверить, удовлетворяет ли введенное значение в ячейку правилам валидации данных на лету. Aspose.Cells API предоставил метод getValidationValue для класса Cell. Если введенное значение в ячейку не удовлетворяет правилам валидации данных, метод getValidationValue для класса Cell возвращает false.
