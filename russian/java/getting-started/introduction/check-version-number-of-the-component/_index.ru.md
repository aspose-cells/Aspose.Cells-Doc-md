{{% alert color="primary" %}} 

В некоторых случаях может возникнуть вопрос, какая у вас версия продукта. Часто мы создаем новые исправления (исправления ошибок для пользовательских сценариев, на которые они указывают) и срочно публикуем их на форумах по их необходимости. Номер версии состоит из основного номера версии, дополнительного номера версии и номера версии исправления. Все определенные компоненты должны быть целыми числами, большими или равными 0. Формат номера версии следующий:

major.minor.hotfix , мы можем увеличить часть на 1 и сделать новую версию. Обычно мы увеличиваем последнюю часть на 1 и создаем новое исправление, чтобы опубликовать его на форумах для пользователей.

В этом документе описываются некоторые способы проверки того, какая версия компонента установлена в вашей системе.

{{% /alert %}} 
## **Проверка номера версии**
### **1) Ручной способ**
Если у вас есть версия/исправление Java (Aspose.Cells for Java), вы можете разархивировать jar-файл библиотеки Aspose.Cells, открыть файл МАНИФЕСТА с помощью блокнота и выполнить поиск строки, т. е. «Specification-Version:», чтобы проверить ее значение.
