{{% alert color="primary" %}} 

Иногда установка Aspose.Cells может вызвать некоторые проблемы с использованием пакета установщика (Aspose.Cells.msi... Установщик Windows) на Windows Vista. В этом документе объясняется, как справиться с этим и успешно установить компонент. Фактически установщику Aspose.Cells нужно создать виртуальную папку на IIS для развертывания его веб-демонстраций (Asp.NET Demos) на вашем компьютере, поэтому перед установкой Aspose.Cells вам нужно иметь права администратора.

{{% /alert %}} 
## **Возможные факторы**
Обычно в Windows Vista продукты/компоненты, которые вы устанавливаете/используете, всегда реализованы с использованием прав "обычного пользователя", даже если вы являетесь администратором. Программам разрешен ограниченный доступ к файловой системе, даже если вы вошли в систему как администратор. Это имеет некоторые негативные побочные эффекты, с которыми вы обычно не сталкиваетесь в Windows XP при входе в систему как администратор.
## **UAC (Контроль учетных записей)**
UAC является частью Windows Vista, которая запрашивает ваше разрешение. Режим **UAC** (также известный как **Режим утверждения администратора**) - это режим работы, который (прежде всего) влияет на работу учетных записей администратора. Когда **UAC** включен (что по умолчанию), вам необходимо явно дать разрешение любой программе, которая хочет использовать "административные" права. Любая программа, которая пытается использовать административные права без вашего разрешения, будет отклонена. **UAC** также требуется для других функций безопасности Windows Vista, включая **Защищенный режим** в Internet Explorer. Защищенный режим Internet Explorer защищает ваш компьютер от недобросовестных веб-страниц и других веб-уязвимостей, включая неизвестные.

Когда **UAC** включен, каждая программа, которую вы запускаете, получает только "стандартный доступ" к системе, даже если вы вошли в систему как администратор. **Windows Vista** имеет встроенную возможность автоматически снижать потенциальные угрозы безопасности в системе. Он делает это, автоматически включая эту функцию, называемую **Контроль учетных записей** (или **UAC** для краткости). **UAC** заставляет пользователей, входящих в группу локальных администраторов, работать так, как будто они были обычными пользователями без административных привилегий. Хотя **UAC** явно улучшает безопасность в **Windows Vista**, в некоторых сценариях вы можете захотеть его отключить, например, при проведении демонстраций перед аудиторией (демонстрации, не связанные с безопасностью, например). Некоторые домашние пользователи могут попытаться отключить **UAC** из-за использования дополнительных ресурсов своей системы.
## **Шаги для успешной установки компонента**
- Пожалуйста, убедитесь, что IIS установлен на вашем компьютере Windows Vista, прежде чем устанавливать **Aspose.Cells**. Это обязательно, потому что установщику Aspose.Cells необходимо создать виртуальную папку на IIS для развертывания веб-демонстраций (Asp.NET Demos).
- Вам нужно отключить **UAC** (Контроль учетных записей). Вы должны убедиться, что у вас есть **Привилегии администратора** с полным управлением системой перед установкой **Aspose.Cells**. В противном случае при установке **Aspose.Cells** с использованием его установщика вы можете получить ошибку # 2869.

Вот несколько способов достичь этого.
### **Используя командную строку**
1. Найдите cmd.exe в вашем каталоге Windows, затем щелкните правой кнопкой мыши и выберите Запустить как... **Администратор**
2. Now, Run the following command on command prompt: msiexec /i <your path>/Aspose.Cells.msi and Enter.
### **Использование Панели управления**
- Нажмите Пуск
- Нажмите Панель управления
- Выберите Учетные записи пользователей и безопасность семьи
- Нажмите Учетные записи пользователей
- Нажмите Включить или отключить управление учетными записями
- Снимите флажок
- Нажмите ОК

{{% alert color="primary" %}} 

Вам нужно будет перезагрузить компьютер, чтобы изменения вступили в силу. Это изменение влияет на все учетные записи на компьютере, а не только на вашу.

{{% /alert %}}
