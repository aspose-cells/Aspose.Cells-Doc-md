## **Установите Aspose.Cells for .NET до NuGet**
NuGet — самый простой способ загрузить и установить Aspose API for .NET. Откройте Microsoft Visual Studio и NuGet менеджер пакетов. Найдите «aspose», чтобы найти нужный Aspose API. Нажмите «Установить», выбранный API будет загружен и указан в вашем проекте.

Примечание. Вы можете посетить веб-страницу nuget aspose.cells для получения дополнительной информации:
[Aspose.Cells for .NET NuGet Пакет](https://www.nuget.org/packages/Aspose.Cells/)

### **Установите Aspose.Cells с помощью графического интерфейса диспетчера пакетов.**
Выполните следующие действия, чтобы сослаться на компонент Aspose.Cells с помощью графического интерфейса диспетчера пакетов:

- Откройте свое решение/проект в Visual Studio.
- Нажмите**Инструменты** -> **Диспетчер пакетов библиотек** -> **Управление пакетами NuGet**из решения. Вы также можете легко получить доступ к той же опции через обозреватель решений. Щелкните правой кнопкой мыши решение или проект и выберите**Управление пакетами NuGet**из контекстного меню.
- Выбирать**В сети**в меню слева и введите «Aspose.Cells» в текстовом поле поиска, чтобы найти пакет Aspose.Cells .NET.
- Нажмите на**Установить**кнопку рядом с записью Aspose.Cells for .NET, чтобы установить последнюю версию в ваш проект.
### **Установите Aspose.Cells с помощью консоли диспетчера пакетов.**
Вы можете выполнить следующие шаги, чтобы сослаться на компонент Aspose.Cells с помощью консоли диспетчера пакетов:

- Откройте свое решение/проект в Visual Studio.
- Выбирать**Инструменты** -> **Диспетчер пакетов библиотек** -> **Консоль диспетчера пакетов**из меню, чтобы открыть консоль диспетчера пакетов.
 Введите команду «Install-Package Aspose.Cells» и нажмите Enter, чтобы установить последнюю полную версию в ваше приложение. В качестве альтернативы вы можете добавить к команде суффикс «-prerelease», чтобы указать, что последний выпуск, включая исправления, также должен быть установлен.
- Вы увидите, что в левом нижнем углу окна появится подсказка «Загрузка Aspose.Cells…», указывающая на то, что загрузка идет.
- После загрузки вы увидите следующие подтверждающие сообщения. Если вы не знакомы с EULA Aspose, рекомендуется прочитать лицензию, указанную в URL-адресе.
- Теперь вы должны обнаружить, что Aspose.Cells успешно добавлен и указан в вашем приложении.
## **Ссылка Aspose.Cells из проекта .NET**
Чтобы использовать Aspose.Cells в приложении, добавьте ссылку на него. Чтобы добавить ссылку с помощью Visual Studio:

1.  в**Обозреватель решений**, разверните узел проекта, на который вы хотите добавить ссылку.
1.  Щелкните правой кнопкой мыши**использованная литература** node для проекта и выберите**Добавить ссылку** из меню.
1.  в**Добавить ссылку** диалоговом окне выберите вкладку .NET (выбрана по умолчанию). Если вы установили с помощью установщика MSI, в верхней панели появится Aspose.Cells.
1.  Выберите его и нажмите**Выбирать**.

Если вы скачали или распаковали только DLL:

1.  Найдите файл Aspose.Cells.dll с помощью**Просматривать** кнопка. Aspose.Cells должен появиться в**Выбранные компоненты** панель диалогового окна.
1.  Нажмите**ХОРОШО** . Ссылка Aspose.Cells появляется под**использованная литература** узел проекта.
### **Ссылка на компонент из проекта VS.NET 2010 Client Profile**
Если целевой платформой вашего проекта является клиентский профиль .NET Framework 3.5/4, используйте файл компонента Aspose.Cells.dll, расположенный в папке net_ClientProfile каталога установки. Например:

-  В**Обозреватель решений** VS.NET 2010 для вашего проекта, щелкните правой кнопкой мыши**использованная литература** и выберите**Добавить ссылку**.
-  Выберите**Просматривать** в диалоговом окне и нажмите «Искать раскрывающийся список».
- Найдите и выберите файл компонента Aspose.Cells.dll в каталоге установки, например: .../Program Files/Aspose/Aspose.Cells for .NET/Bin/net_ClientProfile/ **(Убедитесь, что вы установили продукт с помощью установщика MSI на свой компьютер .)**
-  Нажмите**ХОРОШО** чтобы закрыть диалог

{{% alert color="primary" %}} 

Если целевой платформой вашего проекта VS.NET 2010 является «.NET Framework 4» или другая, просто используйте файл компонента Aspose.Cells.dll, расположенный в папке net4.0/net2.0.

{{% /alert %}} 
## **Ссылка на элементы управления сетью Aspose.Cells из проекта .NET**
Чтобы использовать элемент управления сеткой в приложении, добавьте на него ссылку. Чтобы сослаться на элемент управления сеткой в Visual Studio, выполните следующие действия.

-  В**Обозреватель решений**, разверните узел проекта, на который вы хотите добавить ссылку.
-  Щелкните правой кнопкой мыши**использованная литература** node для проекта и выберите**Добавить ссылку** из меню.
-  в**Добавить ссылку** диалоговое окно, выберите**.NET вкладка** (выбрано по умолчанию). Если вы использовали установщик MSI для установки Aspose.Cells, for .NET, Aspose.Cells.GridDesktop и Aspose.Cells.GridWeb отображаются на верхней панели.
-  Выберите их и нажмите**Выбирать**.
- Ссылки Aspose.Cells.GridDesktop и Aspose.Cells.GridWeb отображаются в узле проекта «Ссылки».

Если вы скачали и распаковали только DLL:

-  Найдите файлы Aspose.Cells.GridDesktop.dll и Aspose.Cells.GridWeb.dll с помощью кнопки Обзор. Aspose.Cells Grid Suite упоминается и должен отображаться в**Выбранные компоненты** панель диалогового окна.
-  Нажмите**ХОРОШО.**
## **Удаление Aspose.Cells for .NET**
Если вы использовали установщик MSI для развертывания Aspose.Cells for .NET, выполните следующие действия, чтобы полностью удалить компонент и элементы управления, связанные демонстрации и документацию:

-  От**Начинать** меню, выберите**Настройки** , с последующим**Панель управления**.
-  Нажмите**Добавить/удалить программы**.
- Выберите Aspose.Cells for .NET (версия).
-  Нажмите**Изменить/удалить** удалить Aspose.Cells.
