{{% alert color="primary" %}} 

Одной из расширенных функций Aspose.Cells.GridWeb является добавление правил проверки ввода для ячеек. Разработчики могут создавать различные типы правил проверки для ячеек, чтобы контролировать и проверять входные значения. В этом разделе обсуждаются поддерживаемые типы проверки и способы их создания.

{{% /alert %}} 
## **Типы валидации**
С помощью Aspose.Cells.GridWeb можно применять три типа проверок:

- Проверка списка.
- Проверка выпадающего списка.
- Проверка пользовательского выражения.

Каждый подробно обсуждается ниже.
### **Проверка списка**
Проверка списка позволяет пользователям вводить данные в ячейку путем ввода или выбора значения из меню. Чтобы создать проверку списка для ячейки:

1. Добавьте элемент управления Aspose.Cells.GridWeb в веб-форму.
1. Доступ к рабочему листу.
1. Получите доступ к ячейке, чтобы добавить проверку.
1. Создайте проверку для ячейки и укажите тип проверки как Список.
1. Добавьте значения для проверки списка.

Пример кода добавляет проверку списка в C1. Когда пользователь щелкает ячейку, появляется список.

1. Добавьте элемент управления Aspose.Cells.GridWeb в веб-форму.
1. Доступ к рабочему листу.
1. Получите доступ к ячейке, чтобы создать проверку.
1. Создайте проверку для ячейки и укажите тип DropDownList.
1. Добавьте значения для проверки.

Пример кода добавляет проверку раскрывающегося списка в C1. Когда пользователь щелкает ячейку, появляется раскрывающийся список, и пользователи могут выбрать из него значение.

1. Добавьте элемент управления Aspose.Cells.GridWeb в веб-форму.
1. Доступ к рабочему листу.
1. Получите доступ к ячейке, чтобы создать проверку.
1. Создайте проверку для ячейки и укажите тип CustomExpression.
1. Установите регулярное выражение проверки.

### **Принудительная проверка**
Используя Aspose.Cells.GridWeb, пользователи могут отправлять входные данные на сервер. Даже если существуют правила проверки для разных ячеек, но для свойства ForceValidation элемента управления GridWeb не задано значение true, неверные входные данные также будут отправлены на сервер, и принудительная проверка не будет выполняться. По умолчанию свойство GridWeb ForceValidation всегда имеет значение true.

 Если для свойства ForceValidation установлено значение true, элемент управления не отправляет данные на веб-сервер до тех пор, пока входные значения всех ячеек не станут допустимыми. Например, если кто-то вводит в ячейку недопустимое входное значение или не вводит значение, активируется проверка на стороне клиента, и пользователи не могут публиковать данные, даже если они нажимают**Представлять на рассмотрение**.
