{{% alert color="primary" %}} 

При создании рабочей книги с большими наборами данных или чтении большого файла Excel Microsoft это, безусловно, потребует больше времени и ресурсов. Общий объем памяти, который займет процесс, всегда вызывает беспокойство. Есть меры, которые могут быть приняты, чтобы справиться с этой проблемой. Aspose.Cells.GridWeb предоставляет некоторые соответствующие параметры и API для уменьшения, сокращения и оптимизации использования памяти. Кроме того, это может помочь процессу работать более эффективно и быстрее. Для рабочего листа, содержащего данные больших ячеек, вы можете загрузить набор данных асинхронно, что может повысить общую производительность для удобства пользователя.

{{% /alert %}} 

Используйте параметр GridWeb.EnableAsync, чтобы оптимизировать память и производительность для данных ячеек. При построении большого набора данных для ячеек. Если вы установите для параметра значение true, загрузка данных будет основываться только на текущей видимой области Windows. Короче говоря, когда вы прокручиваете данные ячеек рабочего листа в GridWeb, он загружает новые данные Windows только на основе текущей позиции прокрутки.
