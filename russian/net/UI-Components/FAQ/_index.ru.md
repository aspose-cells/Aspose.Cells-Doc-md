## **Есть ли ограничения в ознакомительной версии Aspose.Cells Grid Controls?**
 Нет, в ознакомительной версии Aspose.Cells Grid Controls нет ограничений по функциям. Ознакомительная версия предоставляет все функции лицензионной версии, за исключением добавления дополнительного рабочего листа (содержащего**Предупреждение об авторских правах** ) в выходные файлы.
## **На рынке доступно так много элементов управления Grid. Почему я должен купить Aspose.Cells Grid Controls?**
 Что ж, Aspose.Cells Grid Controls имеет очень хорошую цену, чтобы быть доступной для всех типов пользователей. По очень разумной цене он предоставляет вам набор из двух элементов управления для работы с Windows и веб-приложениями. Более того, это не просто Сетка, это ваша**Средство просмотра электронных таблиц, редактор и создатель** в то же время. Вы можете не только связать его с любым источником данных (например, с обычными элементами управления Grid), но также создавать файлы Excel и управлять ими. Он также обеспечивает сильный и богатый**Механизм расчета формулы** для расчета не только встроенных функций (поддерживается Aspose.Cells Grid Controls), но и пользовательских формул, определенных вами. Есть гораздо больше функций пакета Aspose.Cells Grid, которые не могут быть рассмотрены здесь, пожалуйста, обратитесь к странице Типы редакций для получения более подробного списка функций.
## **Недавно я приобрел лицензию на Aspose.Cells Grid Controls. Как я могу использовать эту лицензию в своем приложении с Aspose.Cells Grid Control?**
Пожалуйста, обратитесь к[Лицензирование](/cells/ru/net/licensing/) страница Aspose.Cells Элементы управления сетью. В нем содержится полная информация о том, как использовать лицензию с Aspose.Cells Grid Controls в ваших приложениях.
## **Поддерживаются ли элементы управления Grid Aspose.Cells в Visual Studio.NET 2005?**
Да, элементы управления сеткой Aspose.Cells полностью поддерживаются в Visual Studio.NET 2005 и более поздних версиях.
## **Я использую элемент управления Aspose.Cells.GridWeb на своем веб-сайте с помощью Visual Studio.NET 2005. Но он вообще не работает. В чем проблема?**
 Aspose.Cells.GridWeb поддерживает оба**Файловая система** а также**HTTP** режимы Visual Studio.NET 2005. Если вы все еще не уверены, ознакомьтесь с пошаговым руководством по работе с Aspose.Cells.GridWeb с использованием Visual Studio.NET 2005.
## **Как я могу развернуть свой веб-проект/решение на основе Aspose.Cells.GridWeb на сервере?**
Если вам нужно развернуть веб-приложение с элементом управления GridWeb, вы должны скопировать «acw_client" в папку вашего проекта, по крайней мере, ваше веб-приложение (развернутое на сервере) не сможет его найти. Вы всегда можете указать путь сценария, добавив следующие строки кода в раздел конфигурации (например, в файле web.config в вашем Проект VS.NET)._client» — это папка, содержащая файлы и Aspose.Cells. GridWeb использует эту папку для управления своей внутренней конфигурацией, в ней есть файлы сценариев, файлы изображений и другие файлы для указания поведения GridWeb и настройки других операций. Файл конфигурации используется для предотвращения использование встроенных клиентских ресурсов (изображений, скриптов и т. д.), что полезно в некоторых случаях/сценариях.

**XML**

{{< highlight "csharp" >}}

 <appSettings>

  <add key="aspose.cells.gridweb.acw_client_path" value="/grid/acw_client/"/>

  <add key="aspose.cells.gridweb.force_script_path" value="true"/>

  <add key="aspose.cells.gridweb.forcepath" value="true"/>

 </appSettings>

{{< /highlight >}}

{{% alert color="primary" %}} 

Путь всегда связан с каталогом проекта. Вы не должны использовать какой-либо каталог, который находится за пределами каталога проекта. Поэтому необходимо скопировать каталог «acw_client» (@ваша папка установки GridWeb) в каталог проекта.

{{% /alert %}}
