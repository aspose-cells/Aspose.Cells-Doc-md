Этот документ описывает изменения в общедоступном API Aspose.Cells от версии 8.2.0 до 8.2.1, которые могут быть интересны разработчикам модулей/приложений. В нем содержится информация не только о новых и обновленных общедоступных методах, но и описания любых изменений в поведении за кулисами в Aspose.Cells.

{{% /alert %}} 
## **Добавлен метод GetValidation() для класса Cell**
Проверка данных - одна из функций, которую используют разработчики электронных таблиц, чтобы предотвратить вставку недопустимых значений в определенную ячейку. С Aspose.Cells for .NET 8.2.1 API предоставляет простой механизм, который идентифицирует, была ли применена проверка данных к ячейке. Используйте метод GetValidation класса Cell для получения любой примененной проверки. Если проверки нет, метод возвращает null. Аналогично, вы можете использовать метод GetValidationInCell класса ValidationCollection для получения примененной проверки к любой ячейке, указав ее индексы строки и столбца.

{{% alert color="primary" %}} 

Пожалуйста, ознакомьтесь с подробной статьей по [Получение примененной валидации в ячейке](/cells/ru/net/get-validation-applied-on-a-cell/) для получения дополнительной информации.

## **Добавлен метод GetValidationValue() для класса Cell**
Помимо определения была ли применена валидация, также можно проверить, удовлетворяет ли заданное значение правилам валидации данных для определенной ячейки. Эта функция полезна в ситуациях, когда требуется проверять, удовлетворяет ли введенное значение в ячейке правилам валидации данных на лету. В API Aspose.Cells добавлен метод GetValidationValue для класса Cell. Если значение, введенное в ячейку, не удовлетворяет правилам валидации данных, метод GetValidationValue для класса Cell возвращает false.

{{% alert color="primary" %}} 

Пожалуйста, ознакомьтесь с подробной статьей по [Проверка удовлетворения значения ячейки правилам валидации данных](/cells/ru/net/verify-that-cell-value-satisfies-data-validation-rules/) для получения дополнительной информации.
