## **Upgrading to Aspose.Cells**

Sometimes previous users (e.g., Aspose.Excel users) might encounter an issue or find that some features are missing in the version; alternatively, they need to upgrade to the newer Aspose.Cells for their requirements. It should be noted that Aspose.Excel has been renamed to Aspose.Cells, so we can no longer continue or make amendments with the older versions of the Aspose.Excel component. The new Aspose.Cells component is more reliable, stable, and robust with enhanced and advanced functionality. We specifically focus on optimizing its performance for a wide range of scenarios. Just to emphasize, there is not much effort involved in upgrading to the Aspose.Cells component. However, when you upgrade to it,

**Please note:** 

Some APIs (classes, structs, enumerations, etc.) have been added based on advanced features we now offer. The previous classes (with their members) are retained and optimized, but a few classes are renamed. Some additional members (properties, methods, etc.) are also included for the classes, with very few being excluded.

## **The following is the list of classes that have been renamed:**
1. **Excel** class is replaced by [**Workbook**](https://reference.aspose.com/cells/java/com.aspose.cells/Workbook) class.  
2. **ExcelDesigner** class is replaced by **WorkbookDesigner** class.  
3. **ExcelHelper** class is replaced by **CellsHelper** class.  

For further reference, please check our featured demos: <https://github.com/aspose-cells/Aspose.Cells-for-Java>
