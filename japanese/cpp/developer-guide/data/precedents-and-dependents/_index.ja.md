複雑な財務ワークシート、特に共同で開発されたワークシートでは、最も厄介なエラーが隠れてしまう可能性があります。数式で先行セルと従属セルが使用されている場合、数式の精度をチェックしてエラーの原因を見つけることが困難になることがあります。

{{% /alert %}} 
##  **導入**
- **先行セル**別のセルの数式によって参照されるセルです。たとえば、セル D10 に数式 =B5 が含まれている場合、セル B5 はセル D10 の前になります。
- **依存セル**他のセルを参照する数式が含まれています。たとえば、セル D10 に数式 =B5 が含まれている場合、セル D10 はセル B5 に依存します。

スプレッドシートを読みやすくするために、スプレッドシート上のどのセルが数式で使用されているかを明確に示すことができます。同様に、他のセルの依存セルを抽出したい場合があります。

Aspose.Cells を使用すると、セルをトレースして、どのセルがリンクされているかを見つけることができます。
##  **先行および従属 Cells: Microsoft Excel のトレース**
計算式は、クライアントによる変更に基づいて変更される場合があります。たとえば、セル C1 が数式を含む C3 および C4 に依存しており、C1 が変更される (つまり数式が上書きされる) 場合、ビジネス ルールに基づいてスプレッドシートのバランスを保つために C3 と C4、または他のセルを変更する必要があります。

同様に、C1 に式「=(B1*22)/(M2*N32)」が含まれているとします。 C1 が依存するセル、つまり、前のセル B1、M2、および N32 を見つけたいと考えています。

特定のセルの他のセルへの依存関係を追跡する必要がある場合があります。ビジネス ルールが数式に埋め込まれている場合、依存関係を見つけて、それに基づいていくつかのルールを実行したいと考えます。同様に、特定のセルの値が変更された場合、ワークシート内のどのセルがその変更の影響を受けるでしょうか?

Microsoft Excel を使用すると、前例と依存関係を追跡できます。
