## **症状**

「バージョン Aspose.Cells を使用して、ワークブックを Response オブジェクトに保存しようとするときに Save メソッドを使用すると、このエラーが発生します。このコード スニペットは、オンライン ドキュメントに記載されています。」

### **エラーのスクリーンショット**

### **解決**

使ってください**.NET 2.0**VS.NET 2008/2010 で正常に動作するため、製品のコンパイル済みバージョン。実際には、さまざまな環境、プロジェクトの種類、システムなどに個別の dll を提供しています。参照用に、以下を確認してください。<https://docs.aspose.com/cells/net/using-aspose-cells-on-32-bit-and-64-bit-platforms/>

 Aspose.Cells for .NET は互換性があり、すべての .NET フレームワーク バージョン (2.x、3.x、4.x など) と正常に動作します。 .NET フレームワークのバージョンごとに異なる dll を提供しています。詳細については、**readme.txt**インストールディレクトリの「\Bin」フォルダにあるファイル。でも、これ**readme.txt**ファイルが存在します。

 Webアプリケーションで弊社製品を使用する場合は、**NET 2.0**「/bin」ディレクトリのフォルダ。参考までに、dll は**.NET 3.5 クライアント プロファイル**ディレクトリは、VS.NET のターゲット フレームワークとしてネット フレーム クライアント プロファイルを持つコンソール アプリケーションにのみ使用されます。プロジェクトを確認してください。プロジェクトがこの dll を参照している可能性があります。
