{{% alert color="primary" %}} 

あ*パラメータ化されたレポート*レポートの処理時に使用される入力値を受け入れるレポートです。

パラメータ化されたレポートを使用すると、実行時に設定された値に基づいてレポートの出力を変えることができます。 Reporting Services では、クエリ パラメーターとレポート パラメーターの 2 種類のパラメーターがサポートされています。

- **クエリ パラメータ**データ処理中にデータを選択またはフィルタリングするために使用されます。クエリ パラメーターが指定されている場合は、レポートのデータを取得する SELECT ステートメントまたはストアド プロシージャを完了するために、ユーザーまたは既定のプロパティによって値を指定する必要があります。
- **レポート パラメータ**レポート処理中にデータの別の側面を表示するために使用されます。レポート パラメーターは通常、大量のレコード セットをフィルター処理するために使用されますが、レポート内のクエリや式によっては、他の用途にも使用できます。

レポート パラメーターは、レポートで定義され、レポート サーバーによって処理されるという点で、クエリ パラメーターとは異なります。一方、クエリ パラメーターは、データセット クエリの一部として定義され、データベース サーバーで処理されます。 Aspose.Cells.Report.Designer では、Microsoft Query でのクエリ作成時にクエリ パラメータを指定します。

レポート パラメーターを作成し、クエリ パラメーターを Aspose.Cells.Report.Designer の対応するレポート パラメーターにマップできます。このようにして、レポート パラメーターの値を選択し、それらをクエリに渡して、データ ソースから取得するデータを制限することができます。
