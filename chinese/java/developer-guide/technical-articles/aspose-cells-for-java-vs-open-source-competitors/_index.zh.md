## **概览**

大多数商业场景需要丰富内容的报告，以满足特定决策或任务需求。它们还需要一些数据收集、分析和与数据库系统接口的方式。

常见解决方案是使用电子表格进行数据收集、分析和演示。该解决方案应能够创建完全格式化的Microsoft Excel文件，可分析商业场景，并理想情况下，将这些基本功能扩展到探索捕获和导出数据到其他常用办公应用程序的过程。这些发展可能包括利用Microsoft Excel的内置功能和函数。

如今市场上存在许多承诺提供丰富功能的API进行电子表格管理。在这里，我们对Aspose.Cells for Java与其开源竞争对手JExcelAPI和POI的HSSF进行了功能分析。让我首先简要介绍这三者。

## **Java的Aspose.Cells**

Aspose.Cells for Java 是一个灵活的组件，可以让 Java 应用程序在系统（客户端或服务器）上创建和管理 Excel 电子表格，而无需安装 Microsoft Excel。与其他供应商的类似产品不同，Aspose.Cells for Java 不仅支持电子表格生成和基本文件格式设置功能，还支持许多高级功能。这些高级功能使开发人员可以轻松操纵电子表格内容、单元格格式设置和文件保护选项。

Aspose.Cells for Java 还可以从不同的数据源导入数据到电子表格中，添加常见和复杂的数学、日期/时间、金融、文本公式和函数，导入图片（支持所有主要格式，包括 BMP、GIF、JPG、PNG、WMF、EMF 等），创建注释，创建绘图对象和控制，执行一系列其他任务。

Aspose.Cells for Java 还支持添加插件、VBA 和宏。

### **打开和保存**

除了支持创建或读取本机 Excel 文件等常见功能外，Aspose.Cells for Java 还提供许多有价值的功能，例如保存和打开 Excel 文件（Excel 97 - Excel 2007）到流和从流中导入和导出数据，从 ResultSet 和数组导入和导出数据。

Aspose.Cells 可以导入图表、命名范围、页眉和页脚，并带有 Microsoft Excel 提供的所有属性，插入超链接和图像，并从设计电子表格中导入公式。

### **格式**

工作表数据格式设置很重要，因为它可以改变数据的含义。如果您计划打印工作表、将其发送给客户或向老板展示，您需要考虑是否以查看方便的方式进行格式设置。谨慎使用颜色、底纹、边框、字体、数字格式、对齐、缩进和方向可以使数据从一堆混乱的数据变成易于处理和理解的工作表。

Aspose.Cells for Java 允许轻松地为单元格和单元格范围绘制边框。此外，API 可以应用字体设置（系列和类型、样式、大小、颜色和对齐）并用背景图案填充单元格。API 足够高效，可以格式化完整的行或列，设置对齐方式，换行和旋转单元格中的文本。

Aspose.Cells for Java 支持所有类型的数字格式，包括常规格式、十进制记数中的数字、货币符号、百分比、科学格式、日期/时间值甚至自定义数字格式。

Aspose.Cells 允许开发人员一次性自动调整行和列的大小，并通过方便的 API 配置所有类型的页面设置选项：顶部、左侧、底部、右侧、页眉和页脚边距、方向 - 竖向或横向 - 缩放、纸张大小、打印区域、重复行和列等。

### **独特功能**

Aspose.Cells for Java 还具有许多开发人员只能在其中找到的独特功能，例如支持包括 XLS、XLT、XLSX、CSV、SpreadsheetML、分隔符文本、TXT、XML 和 HTML 在内的广泛格式范围。

API 还允许开发人员将现有工作表的副本（包括全部内容、图像和图表）添加到文件中，通过 API 为图表设置渐变背景，创建注释，设置区域设置，自动筛选和分页符，设置复杂公式，条件格式设置，Microsoft Excel XP 或更高版本中引入的所有类型的保护选项，并操纵命名范围。

此外，Aspose.Cells 添加了自定义图表 API 和高效的公式计算引擎。

## **其他 API**

### **JExcelAPI**

JExcelAPI 是用于读取、写入和修改 Excel 电子表格的 Java API。它是一个开源的 Java API，允许 Java 开发人员读取 Excel 电子表格并动态生成 Excel 电子表格。此外，它包含一个机制，允许 Java 应用程序读取电子表格、修改单元格并写入新电子表格。

在撰写时，它具有有限的一系列功能。它支持：仅以 XLS 文件格式读取和写入原生 Microsoft Excel 文件（Excel97-2003）。它具有一些有限的公式计算支持。它可以操作字体，支持数字和日期格式设置，修改现有工作表，区域设置，保存图表（但不允许开发人员创建或操作图表），插入图像等。

#### **图表支持有限**

JExcelApi 对图表的支持有限：它不支持创建和操作图表。在复制包含图表的电子表格时，只要包含图表的工作表同时包含其他数据，图表就会被写入生成的电子表格中。当复制 Excel 文件时，所有图像信息都将保留，但是在向电子表格中添加图像时，只支持 PNG 格式的图像。

#### **无法复制工作表**

没有 API 可以直接在工作簿内或工作簿之间复制工作表。这个任务可以间接完成，但需要一些工作。例如，使用循环，使用 WritableCell.copyTo() 方法一次复制单元格，这将产生深度复制。然而，格式只是浅复制，所以您需要获取单元格格式，并使用复制构造函数，然后对刚刚复制的单元格调用 WritableCell.setCellFormat()。这是一项很大的工作。

#### **无优化**

另一个限制是 JExcelAPI 不执行优化以减小文件大小，我们可能会看到该组件生成的输出文件大小很大，以 MB 为单位。

JExcelAPI 没有自动调整行或列大小的 API。您需要编写代码来扫描每列中的单元格，计算最大长度，然后调用 WritableSheet.setColumnView() 和 WritableSheet.setRowView() 进行设置。

#### **缺失的特性**

还有一些障碍：API 不支持数据透视表和下拉列表。对于单元格的限制验证选项有限。页面设置和打印选项并没有完全支持，比如重复行和列、并没有支持所有类型的保护选项（包括与密码相关的工作表）。它也不支持数据排序、自动筛选数据、条件格式设置、绘图对象、控件等等许多有价值的功能。

### **POI-HSSF**

HSSF 是 POI 的组件，用于读写 Excel 电子表格。它拥有比 JExcelAPI 更丰富的功能，包括读取和写入原生 Microsoft Excel 文件（尚不支持 Excel97-2003 - XLSX OOXML 文件格式），格式化单元格（数字格式、字体、颜色、边框、对齐等），合并单元格，页面设置选项，导入图像、形状、命名范围，创建批注、页眉和页脚，超链接，自动调整行和列等。

#### **图表支持有限**

对于 POI-HSSF API 还有一些已知的限制。例如，目前无法创建图表。但您可以在 Excel 中创建图表，使用 HSSF 修改图表数据值，然后写出新的电子表格。另一个限制是数据透视表的支持，生成数据透视表不可行。

#### **公式计算**

虽然 POI 的 org.apache.poi.hssf.usermodel 支持公式，但缺乏强大的公式计算引擎。它支持包含单元格引用字符串、整数和浮点文字、相对或绝对引用、算术和逻辑运算符的公式，但不支持数组公式、一元运算符和 3D 引用。

#### **缺失的特性**

HSSF API 也不支持数据透视表。它在单元格上设置的限制数据验证选项有限。它还不支持数据排序和自动筛选数据等功能。
