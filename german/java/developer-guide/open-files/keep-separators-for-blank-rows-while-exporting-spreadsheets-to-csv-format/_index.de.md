## **Behalten Sie Trennzeichen für leere Zeilen bei, während Sie Tabellenkalkulationen in das CSV-Format exportieren**

Aspose.Cells bietet die Möglichkeit, Zeilentrenner beim Konvertieren von Tabellenkalkulationen in das CSV-Format beizubehalten. Hierfür können Sie die verwenden**[KeepSeparatorsForBlankRow](https://reference.aspose.com/cells/java/com.aspose.cells/txtsaveoptions#KeepSeparatorsForBlankRow)**Eigentum von**[TxtSaveOptions](https://reference.aspose.com/cells/java/com.aspose.cells/TxtSaveOptions)**Klasse.**[KeepSeparatorsForBlankRow](https://reference.aspose.com/cells/java/com.aspose.cells/txtsaveoptions#KeepSeparatorsForBlankRow)**ist eine boolesche Eigenschaft. Um die Trennzeichen für Leerzeilen beim Konvertieren der Excel-Datei in CSV beizubehalten, legen Sie die**[KeepSeparatorsForBlankRow](https://reference.aspose.com/cells/java/com.aspose.cells/txtsaveoptions#KeepSeparatorsForBlankRow)**Eigentum zu**Stimmt**.

Der folgende Beispielcode lädt die[Excel-Quelldatei](KeepSeparatorsForBlankRow.xlsx). Es setzt**[TxtSaveOptions.KeepSeparatorsForBlankRow](https://reference.aspose.com/cells/java/com.aspose.cells/txtsaveoptions#KeepSeparatorsForBlankRow)**Eigentum zu**Stimmt** und speichert es als[KeepSeparatorsForBlankRow.out.csv](KeepSeparatorsForBlankRow.out.csv). Der Screenshot zeigt den Vergleich zwischen der Excel-Quelldatei, der beim Konvertieren der Tabelle in CSV generierten Standardausgabe und der durch die Einstellung generierten Ausgabe**[KeepSeparatorsForBlankRow](https://reference.aspose.com/cells/java/com.aspose.cells/txtsaveoptions#KeepSeparatorsForBlankRow)**zu**Stimmt**.
