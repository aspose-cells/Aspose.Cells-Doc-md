## **Sintomo**

"Utilizzando la versione di Aspose.Cells, ottengo questo errore quando uso il metodo Salva cercando di salvare il workbook sull'oggetto Risposta. Trovo questo frammento di codice documentato nella documentazione online."

### **Screenshot dell'Errore**

![todo:image_alt_text](no-overload-for-method-save-takes-4-arguments_1.png)

### **Soluzione**

Please use **.NET 2.0** compiled version of the product as it works fine on VS.NET 2008/2010. Actually we provide separate dll's for different environments, project types and systems etc. For reference, please check:<https://docs.aspose.com/cells/net/using-aspose-cells-on-32-bit-and-64-bit-platforms/>

Aspose.Cells for .NET è compatibile e funziona bene con tutte le versioni del framework .NET cioè 2.x, 3.x, 4.x ecc. per qualsiasi tipo di progetto ad es. Asp.NET/Winforms, progetto Web, Applicazione console o altri progetti ecc. Forniamo diversi dll per diverse versioni del framework .NET. Per ulteriori informazioni, leggi il file **readme.txt** nella cartella "\Bin" della tua directory di installazione. Ma, questo file **readme.txt** è presente.

Quando si utilizza il nostro prodotto in un'applicazione web, utilizzare la Aspose.Cells.dll dalla cartella **NET 2.0** nella directory "/bin". Per la tua informazione, il dll nella directory **NET 3.5 Client Profile** è utilizzato solo per l'applicazione console con profilo client del framework Net come framework target di VS.NET. Controlla il tuo progetto, è possibile che il tuo progetto faccia riferimento a questo dll.
