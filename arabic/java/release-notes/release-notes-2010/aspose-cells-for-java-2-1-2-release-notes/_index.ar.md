 تغييرات ملحوظة للمستخدمين:



في الإصدارات القديمة ، قد تتسبب أساليب Cell.getStyle () / Row.getStyle () / Column.getStyle () في صيانة Cell / Row / Column لكائن النمط الخاص بها. سيؤدي تعديل كائن Style المرتجع لـ getStyle () لاحقًا إلى تغيير نمط Cell / Row / Column مباشرةً.

 من هذا الإصدار ، سيتم تجميع كافة كائنات النمط التي تم تعيينها إلى Cell / صف / عمود لاعتبارات الأداء وسيحتفظ Cell / صف / عمود بمرجع نمط فقط. تعديل كائن النمط الذي تم إرجاعه لـ getStyle () لاحقًا لن يغير نمط Cell / Row / Column. لجعل التعديل ساري المفعول ، يحتاج المستخدمون إلى استدعاء setStyle () لـ Cell / Row / Column بعد تعديل النمط.

هذه القاعدة مطلوبة أيضًا لطرق Style.getFont () / getColor () / getPatternColor () / getBorderColor (). في الإصدارات القديمة ، قد يؤدي تعديل كائن Font / Color المرتجع إلى تغيير النمط مباشرةً. في الإصدار الجديد ، بعد إجراء تغييرات على كائن الخط / اللون ، يحتاج المستخدمون إلى استدعاء Style.setFont () / setColor () / setPatternColor () / setBorderColor () لجعل التعديل ساري المفعول.
